Introducción

Eteba Chat es una plataforma de automatización comercial con IA que permite a cualquier negocio — tiendas, empresas de servicios, restaurantes, agencias — crear asistentes virtuales inteligentes que atienden clientes, gestionan consultas sobre productos/servicios y procesan pedidos automáticamente las 24 horas.

¿Para quién es?

  • Tiendas online y físicas — Automatiza consultas de inventario, precios y pedidos
  • Empresas de servicios — Responde preguntas frecuentes y agenda citas
  • Restaurantes y delivery — Muestra menús y recibe órdenes por chat
  • Cualquier negocio — Si tienes clientes que hacen preguntas, Eteba Chat te ahorra tiempo

¿Cómo funciona?

  1. Regístrate — Crea tu cuenta con Google en segundos.
  2. Configura tu asistente — Define el manual operativo, tono, idioma y catálogo de productos/servicios.
  3. Instala el widget — Una línea de código en tu sitio web.
  4. Tu asistente trabaja 24/7 — Atiende clientes, busca en tu catálogo en tiempo real y genera pedidos.

Inicio Rápido

Pon tu asistente en funcionamiento en menos de 5 minutos.

Paso 1: Crear cuenta

Haz click en "Comenzar Gratis" y autentícate con tu cuenta de Google. Se creará automáticamente tu tenant (espacio de trabajo aislado).

Paso 2: Configurar tu asistente

En el Dashboard → Configuración, define:

  • Nombre — Cómo se presenta tu asistente (ej: "Asistente de MiTienda")
  • Manual Operativo — Instrucciones de comportamiento. Ejemplo:
Eres el asistente virtual de MiTienda. Hablas en español de forma amable.
Cuando te pregunten por productos, busca en el catálogo.
Si un cliente quiere comprar, pídele: nombre, teléfono y ciudad de entrega.
No inventes productos que no están en el inventario.

Paso 3: Instalar el widget

Copia este código y pégalo antes del cierre </body> en tu sitio web:

<script src="https://eteba-chat.onrender.com/widget/widget.js?tenant_id=TU_TENANT_ID"></script>

Reemplaza TU_TENANT_ID con el ID que aparece en Dashboard → API Keys.

Paso 4: Probar

Visita tu sitio web. Verás un botón verde en la esquina inferior derecha. Haz click y prueba una consulta como "¿Qué productos tienen?"

¿Qué sigue?

  • Agrega productos a tu catálogo desde el Dashboard
  • Configura un proxy PHP si usas MySQL externo
  • Ingesta documentos de conocimiento (políticas, FAQ, tarifas) con la API de ingesta

Widget Embebido

El widget es un componente JavaScript autocontenido que se inyecta en cualquier sitio web y proporciona una interfaz de chat flotante.

Instalación

<!-- Pegar antes de </body> -->
<script src="https://eteba-chat.onrender.com/widget/widget.js?tenant_id=TU_TENANT_ID"></script>

Parámetros

ParámetroTipoRequeridoDescripción
tenant_idUUIDSíIdentifica tu negocio. Se obtiene del Dashboard.

Comportamiento

  • Se posiciona en la esquina inferior derecha (configurable por CSS)
  • Carga el CSS automáticamente desde el mismo servidor
  • Detecta automáticamente si está en producción o desarrollo
  • Muestra tarjetas de producto con imagen, precio y botón "Encargar"
  • Responsive — se adapta a móvil y desktop

Personalización CSS

Puedes sobreescribir los estilos del widget usando selectores con mayor especificidad:

/* Cambiar color primario del widget */
.eteba-ai-widget-container {
  --eteba-primary: #8b5cf6; /* Violeta en vez de verde */
}

/* Cambiar posición */
.eteba-ai-widget-container {
  bottom: 100px;
  right: 40px;
}

Eventos

El widget dispara eventos en window que puedes escuchar:

// Cuando el usuario envía un mensaje
window.addEventListener('eteba:message-sent', (e) => {
  console.log('Usuario preguntó:', e.detail.message);
});

// Cuando el bot responde
window.addEventListener('eteba:response', (e) => {
  console.log('Bot respondió:', e.detail.response);
});

API Reference

La API REST de Eteba Chat te permite integrar el asistente en cualquier sistema, app móvil o workflow automatizado.

Base URL

https://eteba-chat.onrender.com

Endpoints

POST /api/query

Envía una consulta al asistente y recibe la respuesta procesada.

POST /api/query
Content-Type: application/json

{
  "tenantId": "e22e9ee0-d29a-4172-88de-fb9ad14c9c1b",
  "prompt": "¿Tienen zapatillas Nike disponibles?"
}

Respuesta exitosa (200)

{
  "type": "SQL",
  "results": [
    {
      "name": "Zapatilla Nike Air Max",
      "price": "45000",
      "stock": 3,
      "description": "Zapatilla deportiva Nike Air Max talla 42",
      "image_url": "uploads/productos/nike-air.jpg"
    }
  ],
  "humanResponse": "¡Sí! Tenemos las Zapatillas Nike Air Max disponibles..."
}

Tipos de respuesta

typeDescripciónCuándo se retorna
SQLResultados de catálogo en tiempo realConsultas de productos, precios, stock
SEMANTICResultados de búsqueda semántica (RAG)Consultas sobre envíos, políticas, FAQ
SALUDO_SOPORTE_GENERALRespuesta conversacionalSaludos, pedidos, soporte general

POST /api/ingest

Ingesta contenido de conocimiento para la búsqueda semántica (RAG). Útil para cargar políticas, FAQ, tarifas de envío, etc.

POST /api/ingest
Content-Type: application/json

{
  "tenantId": "e22e9ee0-d29a-4172-88de-fb9ad14c9c1b",
  "content": "Tarifas de envío: Malabo 2000 CFA, Bata 3000 CFA. Entregas de lunes a viernes."
}

Respuesta exitosa (200)

{
  "success": true,
  "data": { "chunks_processed": 1 }
}

Errores

CódigoSignificadoCausa común
400Bad RequestFalta tenantId o prompt
500Internal ErrorError en la IA, DB o proxy PHP

Webhooks

Recibe notificaciones HTTP en tiempo real cuando ocurren eventos en tu asistente.

Eventos disponibles

EventoDescripciónPayload
order.created Se creó un nuevo pedido Datos del pedido completo
conversation.started Un usuario inició una conversación Timestamp + metadata
product.queried Se consultó un producto del catálogo Nombre del producto + resultado

Configuración

Desde Dashboard → Integraciones → Webhooks, agrega la URL de tu endpoint:

https://tu-servidor.com/webhooks/eteba

Payload de ejemplo (order.created)

{
  "event": "order.created",
  "timestamp": "2024-07-14T12:30:00Z",
  "data": {
    "id": 142,
    "producto_nombre": "Zapatilla Nike Air Max",
    "cliente_nombre": "Juan Ndong",
    "cliente_telefono": "+240 222 334 455",
    "ciudad_entrega": "Malabo",
    "precio_producto": 45000,
    "tienda_id": 1
  }
}

Verificación de firma

Cada webhook incluye un header X-Eteba-Signature con un HMAC-SHA256 del body usando tu webhook secret. Valídalo para asegurar autenticidad:

const crypto = require('crypto');
const signature = req.headers['x-eteba-signature'];
const expected = crypto
  .createHmac('sha256', WEBHOOK_SECRET)
  .update(JSON.stringify(req.body))
  .digest('hex');

if (signature !== expected) {
  return res.status(401).send('Invalid signature');
}

Autenticación

Eteba Chat usa OAuth 2.0 con Google como proveedor de identidad, gestionado por InsForge Auth.

Flujo de autenticación

  1. El usuario hace click en "Iniciar Sesión con Google"
  2. Se redirige a InsForge Auth → Google OAuth consent
  3. Google devuelve un token de acceso
  4. InsForge crea/asocia el usuario y devuelve un access_token
  5. El frontend almacena el token y lo usa para requests autenticados

Uso del token

// El token se envía en el header Authorization
fetch('https://eteba-chat.onrender.com/api/protected-endpoint', {
  headers: {
    'Authorization': 'Bearer TU_ACCESS_TOKEN',
    'Content-Type': 'application/json'
  }
});

Multi-tenant

Cada usuario autenticado tiene un tenant_id (UUID) que aísla sus datos:

  • Los productos, pedidos y conocimiento RAG son exclusivos de cada tenant
  • El widget identifica el tenant por el parámetro tenant_id en la URL del script
  • Las políticas RLS (Row Level Security) en la base de datos garantizan el aislamiento

Ejemplos de Integración

1. Consulta simple con cURL

curl -X POST https://eteba-chat.onrender.com/api/query \
  -H "Content-Type: application/json" \
  -d '{
    "tenantId": "e22e9ee0-d29a-4172-88de-fb9ad14c9c1b",
    "prompt": "¿Qué zapatillas tienen disponibles?"
  }'

2. Integración con JavaScript (frontend)

async function askAssistant(question) {
  const response = await fetch('https://eteba-chat.onrender.com/api/query', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({
      tenantId: 'TU_TENANT_ID',
      prompt: question
    })
  });
  
  const data = await response.json();
  console.log('Respuesta IA:', data.humanResponse);
  
  // Si hay productos, mostrarlos
  if (data.type === 'SQL' && data.results.length > 0) {
    data.results.forEach(product => {
      console.log(`- ${product.name}: ${product.price} CFA (Stock: ${product.stock})`);
    });
  }
  
  return data;
}

// Uso
askAssistant('¿Tienen auriculares?');

3. Integración con Python

import requests

def query_eteba(tenant_id, question):
    response = requests.post(
        'https://eteba-chat.onrender.com/api/query',
        json={
            'tenantId': tenant_id,
            'prompt': question
        }
    )
    data = response.json()
    return data['humanResponse']

# Uso
answer = query_eteba('TU_TENANT_ID', '¿Cuánto cuesta el envío a Bata?')
print(answer)
